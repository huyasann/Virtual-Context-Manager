# VCTX — Virtual Context Manager

> 把本文件内容追加到你的 `CLAUDE.md` 中，或直接放到项目根目录让模型读取。

## 核心规则

你有一个虚拟上下文系统（VCTX），用于突破上下文窗口限制。**必须严格遵守以下协议**。

## 协议

### 1. 压缩前归档（最重要）

**触发条件：** 当你感觉对话已经很长、上下文即将被压缩时。

**立即执行以下步骤：**

**第一步 — 归档当前对话的所有重要内容：**

把本次对话中发生的所有重要事情，按主题拆分成多个 `vctx_archive` 调用。每个调用归档一个独立的主题/任务：

```
vctx_archive(
  title = "一句话描述这个主题",
  content = "完整的技术细节：做了什么、改了哪些文件、关键代码、报错信息、解决方案",
  conclusion = "一句话总结结果",
  keywords = ["关键词1", "关键词2", "关键词3"],
  session_id = "main",
  project_id = "<项目名>"
)
```

**第二步 — 确认归档完成：**

调用 `vctx_list(project_id="<项目名>")` 确认所有块已存入。

**第三步 — 告知用户：**

简短告诉用户："上下文较长，已将本次对话归档到 VCTX。后续可随时检索。"

**注意：**
- 不要遗漏重要内容。宁可多归档一个块，不要漏掉关键决策
- content 必须包含完整技术细节，不要只写摘要
- 每个块聚焦一个主题，不要把所有内容塞进一个块

### 2. 记忆检索

当用户提到之前做过的事情、问"之前那个..."、"上次..."、或上下文被压缩后需要回忆时：

1. `vctx_search(query="<关键词>", project_id="<项目名>")` 搜索
2. 根据结果，`vctx_read(block_id="<id>")` 读取完整内容
3. 基于检索到的内容回答

### 3. 新会话启动

每次新会话开始：

1. `vctx_list(project_id="<项目名>")` 查看已归档内容
2. 目录作为背景知识，**不要主动输出给用户**
3. 用户问到相关话题时再检索具体块

## 归档质量标准

好的归档：
- **title**: 一句话主题，如 "修复登录页面 CORS 错误"
- **content**: 完整代码片段、文件路径、错误信息、解决方案
- **conclusion**: 一句话结论
- **keywords**: 3-6 个关键词

差的归档：
- title 太模糊（"修复 bug"）
- 没有技术细节（"帮用户解决了问题"）
- keywords 太通用（"code", "fix"）

## 关于 CLAUDE.md 与 VCTX 的区别

| | CLAUDE.md | VCTX |
|---|---|---|
| 内容 | 静态规则和偏好 | 动态对话记忆 |
| 谁写 | 用户手动编辑 | 模型自动归档 |
| 何时生效 | 每次会话加载 | 按需检索 |
| 比喻 | 办公室规章制度 | 档案柜里的文件 |

两者互补，不冲突。CLAUDE.md 管"怎么做"，VCTX 管"做过什么"。
